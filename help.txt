Usage : [Argument] (Argument Relevant subarguments)
		
Argument|example : short explaination
	subarguments in order the programm wants to get them. : explaination


Arguments:
##############
-mkAccount : creates an account.
	accountname	|imkerix@live.de 	: The name of the account.
	inboxserver	|pop3.live.com 		: The name of the inboxserver belonging to the account.
	inboxserverport	|22 			: The port of the inboxserver belonging to the account.
	outboxserver	|smtp.live.com 		: The name of the outboxserver belonging to the account.
	outboxserverport|255			: The port of the outboxserver belonging to the account.
-------------
-rmAccount : deletes an account.
	accountname	|imkerix@live.de	: The name of the account to delete.
-------------
-getAccount : Returns a property of an account. 
	accountname	|imkerix@live.de 	: The name of the account you want to know a property from.
	propertyname	|inboxserver		: The name of the property you want to know.

	// If you want to know which accounts exist use getAccount without accountname and propertyname, like this:
	// -getAccount
-------------
-modAccount : Modifies an account you can add or change settings and content with this command.
	accountname	|imkerix@live.de 	: The account to modify.
	propertyname	|outboxserverport 	: The property you want to change.
	new value	|254			: The value you want to set.
###############
###############
-mkMail : creates a new mail.
	accountname	|imkerix@live.de	: The account the new mail will be createt in.
	mailid		|1			: The id of the new mail. (Be careful not to use the id more than once)
-------------
-rmMail : deletes a mail.
	accountname	|imkerix@live.de 	: The account that contains the mail.
	mailid		|1			: The id of the mail.
-------------
-modMail : modifies a mail.
	accountname	|imkerix@live.de	: The account that contains the mail.
	mailid		|1			: The id of the mail.
	propertyname	|senderemail		: The property to change.
	new value	|Imkerix@live.de	: The value to set for the property.

	// If you chose "content" as property the following text can have blank spaces. Like this:
	// -modMail exampleAccount 1 content this is the example text that would be in the mail.
-------------
-getMail : Returns a property of a mail.
	accountname	|imkerix@live.de	: The account that contains the mail.
	mailid		|1			: The id of the mail.
	propertyname	|senderemail		: The name of the property you want to know.

	// If you want to know which mails are in an account use this without id and property, like this:
	// -getMail exampleAccount
